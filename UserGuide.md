# Insurance Complaints App — User Guide

This guide explains how to access and use the app as an end user. It avoids technical/code details.

## Access
- App URL (placeholder): `https://your-app-url`
- API docs (for reference only): `https://your-app-url/api/docs`
- Recommended browsers: latest Chrome, Edge, Firefox, Safari.

## Sign in
- Use the credentials provided by your administrator.
- If multifactor is enabled by your org, complete the MFA step.
- Sessions expire after a period of inactivity; sign in again if prompted.

## User roles (what you’ll see and can do)
- **Admin**: Full access. Can manage users/roles and perform all complaint actions.
- **Complaints Handler**: Create and edit complaints, add communications, progress investigations, and close where allowed.
- **Reviewer**: Review and issue final responses; can edit complaints and add communications; no user management.
- **ReadOnly**: View only; no create/update actions.

## Navigation
- **Dashboard**: High-level counts and quick filters (open, overdue, in investigation, closed).
- **Complaints list**: Search/filter by reference, complainant, status, handler, product, vulnerability, overdue. Click a row to open details.
- **New complaint**: Start a new record with complainant/policy details.
- **Admin** (admins only): Manage users and roles.

## Working a complaint
1) Open a complaint from the list or dashboard.
2) Review summary, timeline, communications, and attachments.
3) Add communications: log inbound/outbound contact; upload attachments if needed.
4) Progress status: acknowledge, move to investigation, draft/issue final response, close. Some steps require Reviewer/Admin.
5) Record outcome and redress (if applicable) before issuing a final response.
6) Reopen: available if the case needs to be reopened (Admin/Reviewer as configured).

## Attachments
- Upload supporting files when adding communications (subject to your org’s policies).
- Download via the attachments section; handle sensitive data per policy.

## Searching and filtering
- Use the complaints list filters for reference, complainant, handler, status, product, vulnerability, overdue.
- Dashboard quick cards switch the list view (open/overdue/in investigation/closed).

## Tips and good practice
- Keep communications and notes up to date for auditability.
- Set correct status promptly to keep SLA views accurate.
- Use meaningful summaries for communications to aid searchability.
- If you lack access to an action, contact an Admin to adjust your role.

## Getting help
- For login or access issues: contact an Admin.
- For process questions: follow your internal complaints handling procedure.
- For outages: raise a ticket with support, providing the app URL and a brief description of the issue.
